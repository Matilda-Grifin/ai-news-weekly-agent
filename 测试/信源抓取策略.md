# 信源抓取策略说明

本文描述当前仓库（`ai_news_skill` + `run_daily_digest`）在 **收集阶段** 的默认策略：哪些走 RSS、哪些走定制站点脚本、哪里用到 **Playwright**。配置入口一般为仓库根目录的 `sources.json`，管线实现在 `project/ai_news_skill/runtime/agent_runtime.py` 的 `collect_stage`。

---

## 1. 总体顺序（收集一条资讯要经过什么）

1. **读配置**：从 `sources.json` 加载信源列表（可经意图规划缩小「本轮要抓的信源」子集）。
2. **站点定制爬虫（优先）**：对每个信源 `name` 在注册表里查找同名爬虫（`project/ai_news_skill/crawlers/site/registry.py`）。  
   - 若命中：执行该爬虫，得到条目；该信源 **`name` 会加入已站点爬取集合**，后续 **不再用 RSS 抓这一条信源**。
3. **RSS / Atom**：对 **未** 被站点爬虫接管的信源，用 `rss_url` 拉取 Feed，`urllib` 下载 XML，`xml.etree` 解析 RSS 2.0 / Atom（实现见 `project/run_daily_digest.py` 的 `collect_news` / `parse_rss`）。
4. **合并去重**：站点结果与 RSS 结果合并后 `dedupe_items`（以 `link` 或标题为键）。
5. **（可选）GNews**：若开启且配置了 `GNEWS_API_KEY`，按意图关键词调用 GNews HTTP API，结果再并入列表。
6. ~~**（已移除）Google HK**~~：曾用 Playwright 打开 Google 香港 SERP，易触发风控；已从主流程删除。

默认开关：`site_crawlers_enabled` 为真时使用定制爬虫；Streamlit 侧未单独关断时与之一致。

**名称匹配规则**：注册表用信源 `name` 与爬虫类的 `name` **不区分大小写**比对，须与 `sources.json` 里字符串一致（含括号、空格），例如 `Anthropic Blog(OpenRSS)` 必须完全一致。

---

## 2. 仅走 RSS / Atom 的信源（当前 `sources.json`）

下列信源 **没有** 在 `registry` 中注册同名爬虫，因此 **只走 `rss_url`**：

| 信源 `name` | Feed 类型 | `rss_url` 说明 |
|-------------|-----------|----------------|
| **arXiv cs.AI** | RSS | arXiv 官方 cs.AI 频道 |
| **腾讯混元GitHub动态** | Atom | GitHub commits feed |
| **阿里通义GitHub动态** | Atom | GitHub commits feed |

**怎么抓**：对每个信源发起 HTTPS GET → 解析 `item` / `entry` → 取标题、链接、摘要、`pubDate`/`updated` → 按 `window_hours` 过滤时间窗口 → 每条带上 `sources.json` 里的 `category`。

---

## 3. 定制站点爬虫（有 RSS 配置也会被「站点优先」覆盖）

以下信源在 `sources.json` 里 **仍填了 `rss_url`（备用/文档用途）**，但只要 `name` 与爬虫一致，**实际采集走定制脚本，不会再用该条的 RSS**：

| 信源 `name` | 实现文件 | 列表页 URL（主要入口） | 列表怎么抓 | 详情怎么抓 |
|--------------|----------|------------------------|------------|------------|
| **OpenAI Blog** | `crawlers/site/openai_news.py` | `https://openai.com/news` | **Playwright**：打开列表页、滚动、尝试点 Load more，再 **BeautifulSoup** 抽文章链接 | **Playwright** 打开文章页 + **BS** 抽标题/时间/描述/正文片段 |
| **Anthropic Blog(OpenRSS)** | `crawlers/site/anthropic_news.py` | `https://www.anthropic.com/news` | **Playwright**：滚动 + 「See more」类按钮 | **Playwright** 详情 + **BS** |
| **Hugging Face Blog** | `crawlers/site/huggingface_blog.py` | `https://huggingface.co/blog` | **Playwright** 滚动列表 | **Playwright** 详情 + **BS** |
| **TechCrunch AI** | `crawlers/site/techcrunch.py` | `https://techcrunch.com/tag/artificial-intelligence/` | 先 **`urllib` 拉列表 HTML** + **BS**；若链接过少则 **Playwright** 再滚列表补链 | **`urllib` + BS** 解析文章页（meta/time/h1/正文） |
| **VentureBeat AI** | `crawlers/site/venturebeat.py` | `https://venturebeat.com/category/ai/` | 先 **`urllib` 列表**；空则 **Playwright** 滚动；非空也常再用 **Playwright** 滚动以应对懒加载 | **`urllib` + BS** 文章页 |

定制爬虫共性：

- 按 `per_source`（通常与配置 `limit` 一致）控制每个信源最多保留条数。
- 按 `window_hours` 过滤发布时间（解析失败的条目可能被丢弃）。
- 输出字段与 RSS 条目对齐，便于后续去重、意图过滤、LLM 富化。

## 4. Playwright 使用一览（按功能）

| 用途 | 位置 | 说明 |
|------|------|------|
| **OpenAI 列表 + 详情** | `openai_news.py` | 列表滚动/Load more；每篇详情页 |
| **Anthropic 列表 + 详情** | `anthropic_news.py` | 列表滚动 + See more；详情页 |
| **Hugging Face 列表 + 详情** | `huggingface_blog.py` | 列表滚动；详情页 |
| **TechCrunch 列表（补全）** | `techcrunch.py` | 仅当静态 HTML 链接不足时用 Playwright 滚列表；**详情仍为 urllib** |
| **VentureBeat 列表** | `venturebeat.py` | 常与 urllib 组合，懒加载时用 Playwright 滚列表；**详情仍为 urllib** |

**无头模式**：站点定制爬虫默认 `chromium.launch(headless=True)`。可通过环境变量 **`SITE_CRAWLER_HEADLESS=false`** 改为有头浏览器（见 `crawlers/site/base.py` 的 `playwright_site_headless()`），便于偶发验证页人工点一下。

**依赖**：需安装 `playwright` 包并执行 `python -m playwright install chromium`。

---

## 5. 与 `external/FinnewsHunter` 的关系

`external/FinnewsHunter` 内的 requests + BeautifulSoup、`needs_js`、增强正文抽取等 **未自动接入** 本管线。当前「非 RSS」能力以 **`project/ai_news_skill/crawlers/site/`** 下各站脚本为准；若要复用 FinnewsHunter 的规则需单独对接或迁移。

---

## 6. 注册新定制爬虫时要做的

1. 在 `crawlers/site/` 新增爬虫类，设置与 `sources.json` **完全一致**的 `name`。
2. 在 `crawlers/site/registry.py` 的 `all_crawlers()` 中注册实例。
3. 在 `sources.json` 中为该信源保留同名 `name` 与合法 `rss_url`（作为未启用站点爬虫或失败时的文档/回退参考；启用站点爬虫后该信源默认不走 RSS）。
